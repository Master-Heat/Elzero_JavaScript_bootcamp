Make sure to use Live Server or any local server to run this project instead of opening the HTML file directly. Browsers block ES module imports when loaded via file:// for security reasons.

CORS (Cross-Origin Resource Sharing): A browser security mechanism that controls whether a web page from one origin can request data from another origin (cross-origin data exchange).

ES Modules: When you use type="module" or import, the browser fetches the module files over the network (even locally).

TL;DR
CORS is a browser security system that restricts cross-origin requests.
ES Modules always fetch files via network requests.
Opening files with file:// fails because no server is present to handle requests or provide CORS headers.
